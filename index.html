¿Qué es un archivo .gitignore?

El archivo ".gitignore" se utiliza para excluir archivos y carpetas que no deben ser seguidos por Git. Esto es útil para evitar incluir archivos temporales, de configuración local o innecesarios en tu repositorio.

Pasos para Configurar un Archivo .gitignore:

  • 1. Crear el archivo: En la raíz de tu proyecto, crea un archivo llamado ".gitignore"
  • 2. Agregar Reglas: Especifica qué archivos o carpetas deseas ignorar. Ejemplos:
    • "*.log" Ignora todos los archivos con extensión "*.log"
    • "node_modules/" Ignora la carpeta "node_modules/"
    • ".DS_Store" y "Thumbs.db" ignora archivos temporales de sistemas operativos.
  • 3. Guardar y Verificar: Guarda el archivo y verifica los archivos ignorados con "git status --ignored".
  • 4. Actualizar el Repositorio: Si hay archivos ya en el repositorio que deberían ser ignorados, elimínalos con "git rm --cached <archivo>" y realiza un commit.
